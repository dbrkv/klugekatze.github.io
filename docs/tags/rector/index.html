Symfony routing from yaml to attributes

It’s Over 9000! Given: Symfony application with 99 controllers and 457 unique route definitions in YAML. PhpStorm Symfony plugin helps a lot with navigation but still have limited refactoring capabilities. When all definitions are in one place it’s simply better. With new PHP Attributes syntax, decided to refactor YAML routing to attributes for easier maintenance. Each route definition have minimum 2 config parameters: path and name. Many routes have methods, _defaults and requirements defined, some contain options parameter.
